## 核心原则：令牌不是消耗品，请勿频繁获取

**重要提醒**：访问令牌(`accessToken`)和刷新令牌(`refreshToken`)设计为**长期有效**的凭证，**不需要**每次调用接口前都重新获取！

## 为什么不要频繁获取令牌？

### **平台角度**

资源浪费：每次获取令牌都会在服务器创建新记录，占用内存

性能影响：频繁调用增加服务器负载，影响所有用户

安全隐患：异常频繁获取可能被识别为攻击行为

### **开发者角度**

效率低下：每次调用多一次网络请求，降低应用性能

配额浪费：可能耗尽API调用配额

用户体验：不必要的延迟影响用户操作流畅度

### **技术角度**

令牌类型

有效期

设计目的

访问令牌

7天/24小时

**多次API调用**，无需重复获取

刷新令牌

30天/7天

**长期维持会话**，减少用户重复授权
