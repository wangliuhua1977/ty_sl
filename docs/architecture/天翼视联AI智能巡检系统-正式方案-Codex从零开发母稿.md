# 天翼视联 AI 智能巡检系统正式方案（Codex Windows 桌面版从 0 开发母稿）

> 版本：V1.0  
> 用途：作为 **ChatGPT 新对话 + Codex Windows 桌面版** 的首轮开发母稿  
> 适用方式：**新建项目从 0 开发，不在原项目上修改**  
> 文档定位：产品方案 + 系统方案 + 开发边界 + 模块拆解 + 策略设计总稿

---

# 1. 文档目标

本文档用于指导重新创建一个全新的“天翼视联 AI 智能巡检系统”。

该系统不是对旧项目做增量改造，而是：

- 新建仓库
- 新建解决方案
- 新建工程结构
- 新建 UI 体系
- 新建业务流程
- 新建数据模型

但必须继承旧项目已经冻结的技术骨架：

- .NET 8 Windows 桌面
- 天翼视联能力开放平台
- 高德地图
- 本地轻量持久化
- 企业微信 / 量子密信 Webhook
- XXTea + HMAC-SHA256 + RSA 的平台通信规则
- GCJ-02 坐标展示规则

本方案的目标是让 Codex Windows 桌面版可以基于本稿，从 0 开始分阶段实现完整系统。

# 2. 项目定位

## 2.1 项目名称

**天翼视联 AI 智能巡检系统**

## 2.2 项目形态

Windows 桌面端运维巡检系统

## 2.3 项目核心定位

以 **地图为中心**、以 **Job 定时巡检** 为驱动、以 **点位可播放性判断** 为核心结果、以 **故障闭环与责任派单** 为抓手、以 **点位体检与智能档案** 为基础、以 **报表沉淀与模拟故障** 为支撑的智能巡检系统。

## 2.4 一句话定义

**系统最终要回答的问题是：该点位当前是否能够正常播放，并在异常发生后完成通知、处理、复检、恢复的闭环。**

# 3. 建设原则

## 3.1 新建工程，不改旧项目

本项目采用全新工程实施：

- 不在旧项目代码上直接修改
- 不以旧 UI 为基础重构
- 不复制旧业务流程作为最终流程
- 允许页面、菜单、导航、模块命名全部重做

## 3.2 技术同构，业务重做

允许重做：

- 界面布局
- 菜单结构
- 页面数量
- 交互方式
- 模块命名
- 流程编排
- 视觉风格
- 数字人交互表现

禁止偏离的技术骨架：

- .NET 8 Windows 桌面
- 天翼视联开放平台接入方式
- 用户无感知 Token 模式
- XXTea/HMAC-SHA256/RSA 规则
- 高德地图承载
- GCJ-02 坐标展示
- 本地轻量持久化
- 企业微信 / 量子密信 Webhook 边界

## 3.3 平台通信原则

平台接入必须遵循：

- HTTPS
- POST
- application/x-www-form-urlencoded
- UTF-8 编码
- 请求头传 `apiVersion=2.0`
- 私有参数加密为 `params`
- 签名为 HMAC-SHA256
- `version=1.1` 时返回数据采用 RSA 解密
- `version=v1.0` 时返回数据采用 XXTea 解密

## 3.4 配置与凭证管理原则

真实配置不得写死在源码中：

- AppId
- AppSecret
- RSA 私钥
- 高德 Key
- 企业微信 Webhook
- 量子密信 Webhook
- enterpriseUser / parentUser

应统一外置并通过本地配置注入。

# 4. 实际业务场景与系统目标

## 4.1 业务场景

本系统面向“慢直播点位运维巡检”。

现场核心问题不是普通安防事件研判，而是：

- 设备是否在线
- 直播是否能正常打开
- 画面是否黑屏、模糊、遮挡、偏斜、低照度、冻结
- AI 画面异常是否真实存在
- 发现故障后能否派单处理并持续跟踪恢复

## 4.2 系统目标

### 目标 1：自动发现问题
通过接口和视频能力发现：

- 在线异常
- 普通设备告警
- AI 告警
- 视频不可播放
- 画面异常

### 目标 2：统一形成结论
对每个点位输出统一的 **可播放性判断**。

### 目标 3：完成故障闭环
故障发现后，系统能自动：

- 派单通知
- 进入处理中
- 定时复检
- 恢复销警
- 沉淀报表

### 目标 4：形成点位长期资产
为每个点位建立长期可持续积累的智能档案。

### 目标 5：支持展示、培训、验收
提供模拟故障与数字人巡检动画能力。

# 5. 系统核心业务主线

系统固定主线如下：

**接口判定 → 点位体检 → 可播放性判断 → 截图/播放复核 → 故障派单 → 定时复检 → 恢复销警 → 报表沉淀**

说明：

- 接口判定负责快速批量发现问题
- 点位体检负责提前固化播放策略
- 可播放性判断是统一结果语言
- 截图/播放复核用于人工确认
- 故障派单与复检构成闭环
- 报表用于管理与复盘

# 8. 巡检范围配置中心

## 8.1 功能定义

管理员可以指定系统的巡检点位范围，并将该范围持久化保存，供后续所有 Job、地图展示、结果、故障、报表统一复用。

## 8.2 支持的配置来源

- 按接口目录筛选纳入
- 按单独点位纳入
- 按排除规则剔除点位

## 8.3 生效公式

**最终巡检范围 = 纳入目录下的全部点位 + 单独纳入点位 - 单独排除点位**

## 8.4 多套巡检范围方案

系统支持长期维护多套巡检范围方案。

建议内置支持：

- 默认巡检范围
- 全量巡检范围
- 重点点位巡检范围
- AI 专项巡检范围
- 故障复检范围
- 演示/测试巡检范围

## 8.5 每套方案持久化字段

- 方案名称
- 方案类型
- 是否启用
- 纳入目录列表
- 单独纳入点位列表
- 单独排除点位列表
- 最终生效点位数量
- 备注
- 创建时间
- 修改时间
- 修改人

## 8.6 相关展示 UI 联动规则

地图及相关展示 UI 默认只显示当前巡检范围方案内的点位。

与范围方案联动的界面包括：

- 地图巡检总台
- 巡检结果中心
- 截图总览与人工复核中心
- 点位智能档案列表
- 故障闭环中心
- 报表中心

## 8.7 地图切换规则

- 系统启动默认加载当前启用范围方案
- 支持手工切换范围方案
- 切换后地图点位、统计、热区、右侧摘要同步刷新
- 支持管理员查看“全量设备对照视图”校验范围方案是否正确

# 9. 地图与坐标治理设计

## 9.1 地图承载

系统以高德地图为承载。

## 9.2 坐标展示规则

地图统一使用 **GCJ-02** 展示坐标。

若接口返回 BD-09，则展示前先转换为 GCJ-02。

## 9.3 坐标来源

点位坐标可能来自：

- 平台接口坐标
- 本地导入补录坐标
- 管理员人工地图点选坐标
- 无坐标

## 9.4 坐标优先级

正式冻结为：

**管理员人工指定坐标 > 接口坐标 > 其他本地补录坐标 > 未定位**

## 9.5 管理员地图点选人工坐标

管理员可在地图上直接为点位指定坐标：

流程：

- 选择点位
- 点击“手工指定坐标”
- 地图进入拾取模式
- 鼠标点击地图目标位置
- 弹出确认框
- 保存经纬度并立即刷新地图

## 9.6 人工坐标记录字段

- deviceCode
- manualLongitude
- manualLatitude
- sourceType（管理员地图点选 / 导入补录）
- remark
- updatedAtUtc
- updatedBy

## 9.7 地图图层建议

- 在线状态层
- 播放状态层
- 故障状态层
- 健康度层
- AI 异常层
- 重点关注点位层
- 未定位点位层
- 坐标来源层

# 10. 点位体检中心

## 10.1 功能目标

为每个点位提前建立“播放处方”，避免每次点击播放都临时试协议、试参数、试兼容性。

## 10.2 体检结论定义

点位体检固化的是：

- 最优播放策略
- 备用播放策略
- 兼容性结论
- 起播性能结论

不是固化某个长期可用的流地址。

## 10.3 体检类型

- 全量体检
- 定点体检
- 周期复检体检

## 10.4 体检输入

- 设备在线状态
- 点位类型
- 候选流协议
- 编码类型
- 最近播放表现
- 最近失败记录

## 10.5 体检输出字段

每个点位体检档案至少保存：

- deviceCode
- 最优协议
- 备用协议
- videoEnc
- 最近首帧耗时
- 最近起播成功率
- 最近失败原因
- 推荐 playerType
- 推荐 wasm
- 推荐 mute
- 推荐 mediaType
- 最近一次体检时间
- 地址有效期参数
- 是否需复检
- 当前体检等级

## 10.6 体检流程

1. 拉取候选流能力  
2. 校验播放器兼容性  
3. 短时试播  
4. 记录首帧时长与成功率  
5. 生成最优方案与备用方案  
6. 写入点位体检档案

## 10.7 点位播放健康等级

正式固定为：

- A：可秒开播放
- B：可正常播放
- C：可播但较慢
- D：可播不稳定
- E：当前不可播放

# 11. 点位可播放性判断

## 11.1 核心目标

系统所有播放相关能力，最终都要收敛到一个明确结果：

**该点位当前能否正常播放。**

## 11.2 判断层级

### 第一层：接口侧判断

- 设备是否在线
- 是否存在普通告警
- 是否存在 AI 异常
- 是否可获取直播地址

### 第二层：体检侧判断

- 最优协议是否已固化
- 编码是否兼容
- 最近首帧耗时是否正常
- 最近起播成功率是否正常

### 第三层：实际播放判断

- 是否成功起播
- 首帧时间是否可接受
- 是否黑屏
- 是否无流无画
- 是否自动切备流成功

## 11.3 最终统一输出

- 可正常播放
- 可播放但体验较差
- 当前不可播放
- 需重新体检
- 需人工复核

# 13. AI 告警策略中心

## 13.1 功能定义

用户可自定义：

- 哪些 AI 告警类型纳入巡检
- 哪些 AI 告警类型需要自动派单
- 哪些 AI 告警类型只记录不派单

## 13.2 支持两级配置

- 全局配置
- 点位级配置

## 13.3 优先级规则

**点位级配置优先于全局配置。**

## 13.4 每种 AI 告警可配置项

- 是否启用
- 是否纳入巡检
- 是否需要派单
- 是否仅记录不通知
- 默认处理时长
- 默认复检策略
- 默认通知级别

## 13.5 点位级覆盖配置

支持针对某个点位 + 某种 AI 告警类型单独设置：

- 是否启用
- 是否派单
- 是否静默
- 单独处理时长
- 单独责任归属覆盖

## 13.6 AI 告警策略举例

- 画面异常巡检：启用 + 派单
- 区域入侵：启用 + 派单
- 人群聚集：启用 + 人工复核后派单
- 客流统计：启用 + 不派单，仅统计

# 14. 派单触发策略中心

## 14.1 功能定义

管理员可配置故障的派单方式，决定某类故障是：

- 满足条件后自动派单
- 先进入人工复核，确认后再派单

## 14.2 支持的两种模式

### 自动派单模式

适用于：

- 设备离线
- 点位不可播放
- 连续多轮巡检异常
- 高优先级重点点位故障
- AI 画面异常证据充分

### 人工复核后派单模式

适用于：

- 可播放但画面疑似异常
- 模糊 / 遮挡 / 偏斜等轻故障
- AI 告警误报率较高的类型
- 非重点点位的一般异常

## 14.3 派单策略配置层级

支持：

- 全局默认策略
- 批量策略
- 单独策略

## 14.4 派单策略优先级

**单独设置 > 批量设置 > 全局默认策略**

## 14.5 批量设置维度

可按以下维度批量设置自动还是人工派单：

- 区域
- 单位
- 点位分组
- 故障类型
- AI 告警类型
- 重点 / 普通点位等级
- 巡检范围方案

## 14.6 单独设置维度

可针对：

- 单个点位
- 单个点位 + 单故障类型
- 单个点位 + 单 AI 告警类型

设置为：

- 自动派单
- 人工复核后派单
- 仅记录不派单

# 15. 重点关注点位策略中心

## 15.1 功能定义

系统支持管理员将某些点位设置为“重点关注点位”，并为其匹配更严格、更优先的专属策略。

## 15.2 重点关注点位的核心价值

- 更高频巡检
- 更快预加载
- 更快自动派单
- 更短处理时长
- 更短复检周期
- 更强通知提醒
- 地图高亮显示
- 报表单独统计

## 15.3 重点策略内容

### 告警策略
- 哪些 AI 告警启用
- 哪些 AI 告警需要派单
- 哪些故障自动派单
- 是否使用更敏感画面异常判定

### 巡检策略
- 更短巡检周期
- 更高截图频率
- 更高体检频率

### 复检策略
- 更短处理时长
- 更短复检间隔
- 更早超时催办

### 通知策略
- 是否优先通知
- 是否抄送额外管理群
- 是否夜间仍强提醒

### 展示策略
- 地图高亮
- 列表置顶
- 异常摘要单独显示
- 报表单独统计

## 15.4 优先级

**重点点位单独策略 > 单点位普通策略 > 批量策略 > 全局默认策略**

# 16. 视频播放优化与预加载策略

## 16.1 目标

实现：

**接口先筛点，后台预取，点击即播。**

## 16.2 原则

不做全量高频刷流，只对热点点位做智能预刷新。

## 16.3 预刷新对象

- 当前地图视野内点位
- 当前异常点位 TOP N
- 待人工复核点位
- 用户最近点过的点位
- 重点关注点位

## 16.4 预刷新策略

- 到期前静默刷新
- 预拉首帧
- 保留最近成功播放上下文
- 保留主流与备流信息

## 16.5 点位画像驱动的轮询加载策略

系统按每个点位的体检资料参数，动态生成不同的轮询与预加载策略。

影响因子包括：

- 点位优先级
- 地址时效
- 首帧耗时
- 成功率
- 风险等级
- 是否重点点位
- 是否待复核

## 16.6 三层策略模型

### A 类：秒开点位

适用于：
- 异常复核
- 当前视野重点点
- 重点关注点位

策略：
- 静默刷新
- 预加载首帧
- 保持最近播放会话

### B 类：准秒开点位

适用于：
- 区域重点点位
- 历史问题点位

策略：
- 保留最新地址
- 到期前刷新
- 不长期保活播放器

### C 类：普通点位

适用于：
- 全量普通点位

策略：
- 仅保留体检档案
- 点击时按最优方案快速起播

# 22. 故障闭环中心

## 22.1 功能目标

建立完整故障链路：

**故障发现 → 通知派单 → 处理中 → 定时复检 → 恢复销警 → 闭环归档**

## 22.2 故障状态机

每个故障固定状态：

- 新发现
- 已通知
- 处理中
- 待复检
- 已改派
- 已升级
- 已恢复
- 已关闭
- 已忽略

## 22.3 处理宽限期

系统派单后，应给维护人员合理处理时间，不允许短时间重复轰炸通知。

支持：

- 按故障类型默认处理时长
- 管理员手工覆盖处理时长
- 按点位自定义处理时长
- 指定明确复检时间

## 22.4 自动复检

到达处理时限后，系统自动再次巡检该点位：

- 是否在线
- 是否可正常播放
- AI 异常是否消失
- 画面是否恢复

## 22.5 恢复销警

若复检正常：

- 状态改为已恢复
- 记录恢复时间
- 可发送恢复通知
- 进入闭环归档

## 22.6 未恢复升级

若复检仍异常：

- 进入下一轮处理中 / 待复检
- 根据策略决定是否再次通知
- 必要时升级处理

## 22.7 管理员忽略机制

管理员可将某点位或某故障设置为：

- 临时忽略
- 定时忽略
- 长期忽略

忽略后：

- 继续保留记录
- 暂停通知
- 暂停派单
- 仍可选择保留复检或暂停复检

## 22.8 故障静默窗口

支持在某时间段内：

- 故障仍记录
- 故障仍复检
- 不向群外发通知

适用于：

- 夜间
- 施工期
- 已知处理中
- 计划停机窗口

- 支持忽略与静默

- 支持自动与人工派单策略配置
- 支持重点关注点位

- 地图为中心
- 数字人不喧宾夺主
